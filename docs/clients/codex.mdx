# Codex CLI 与 Desktop 配置教程

本教程说明如何使用 SurplusToken API Key 配置 Codex CLI 与 Codex Desktop，以及如何通过 Cockpit Tools 在多个 Codex 账号之间切换。

<SystemSelector variant="content" />

:::warning 密钥安全
API Key 等同于账号密码。不要把密钥发送给他人，也不要写入项目源码、提交到 Git，或出现在公开截图和日志中。

本文已将默认模型配置为 `gpt-5.6-sol`，推理强度配置为 `high`。其中 `<SURPLUSTOKEN_API_KEY>` 是密钥占位符，请替换为你自己的 API Key。
:::

## 开始前准备

开始前，请先完成以下操作：

1. 登录 [SurplusToken](https://surplustoken.com)。
2. 在平台中[创建 API 密钥](/platform/create-api-key)。
3. 确认密钥绑定的分组支持 `gpt-5.6-sol` 模型。

SurplusToken 的公网 API Base URL 为：

```text
https://surplustoken.com/v1
```

如果平台页面提供了校内专用地址，请以页面显示的地址为准。

## 安装 Codex

<SystemContent system="windows">

### Windows

可以从 [Codex 官方页面](https://openai.com/zh-Hans-CN/codex/)安装 Codex Desktop。

如果主要使用命令行，推荐在 WSL2 中安装 Codex CLI。以管理员身份打开 PowerShell 或 Windows Terminal：

```powershell
wsl --install
```

安装完成并重启电脑后，打开 Ubuntu，在 WSL 终端中运行：

```bash
curl -fsSL https://chatgpt.com/codex/install.sh | sh
codex --version
```

后续文档中以 `~/.codex` 开头的路径，均指 WSL 用户目录中的路径，而不是 Windows 用户目录。

</SystemContent>

<SystemContent system="macos">

### macOS

打开 Terminal，运行：

```bash
curl -fsSL https://chatgpt.com/codex/install.sh | sh
codex --version
```

也可以使用 Homebrew：

```bash
brew install --cask codex
```

Codex Desktop 可以从 [Codex 官方页面](https://openai.com/zh-Hans-CN/codex/)安装。

</SystemContent>

<SystemContent system="linux">

### Linux

打开终端，运行：

```bash
curl -fsSL https://chatgpt.com/codex/install.sh | sh
codex --version
```

Linux 用户请使用 Codex CLI。

能够正常显示 Codex 版本号，即表示安装完成。

## 找到配置目录

Codex 主要读取两个文件：

| 运行环境       | 主配置                             | API Key                          |
| -------------- | ---------------------------------- | -------------------------------- |
| Windows 原生   | `%USERPROFILE%\.codex\config.toml` | `%USERPROFILE%\.codex\auth.json` |
| Windows + WSL2 | `~/.codex/config.toml`             | `~/.codex/auth.json`             |
| macOS          | `~/.codex/config.toml`             | `~/.codex/auth.json`             |
| Linux          | `~/.codex/config.toml`             | `~/.codex/auth.json`             |

<SystemContent system="windows">

使用 WSL2 时，请在 WSL 终端创建配置目录：

```bash
mkdir -p ~/.codex
chmod 700 ~/.codex
```

如果使用原生 Windows 环境，可以按 `Win + R`，输入：

```text
%USERPROFILE%\.codex
```

目录不存在时，请先创建 `.codex` 文件夹。

</SystemContent>

<SystemContent system="macos">

在 Terminal 中运行：

```bash
mkdir -p ~/.codex
chmod 700 ~/.codex
```

</SystemContent>

<SystemContent system="linux">

在终端中运行：

## 配置 `config.toml`

打开 `config.toml`，写入以下内容：

```toml
model_provider = "surplustoken"
model = "gpt-5.6-sol"
model_reasoning_effort = "high"
disable_response_storage = true

[model_providers.surplustoken]
name = "SurplusToken"
base_url = "https://surplustoken.com/v1"
wire_api = "responses"
requires_openai_auth = true
```

以上示例已经设置好模型与推理强度，只需按后文配置 API Key 即可使用。

:::info
`base_url` 应填写到 `/v1`，不要写成 `/v1/responses`。Codex 会自动拼接 Responses API 路径。

如果使用平台提供的校内专用地址，只替换 `base_url` 的值，其他字段保持不变。
:::

:::tip 配置放置位置
`model_provider`、`model`、`model_reasoning_effort` 和 `disable_response_storage` 不必占据文件的物理前四行，但必须放在文件中第一个 `[表名]` 之前；本例的第一个表是 `[model_providers.surplustoken]`。

这是因为 TOML 从 `[model_providers.surplustoken]` 开始进入该表；写在它后面的普通字段会被当成该表的成员，而不再是 Codex 的顶层配置。上述四个顶层字段彼此之间的顺序没有要求。
:::

Windows + WSL2 用户首次使用时，还可以在文件中第一个 `[表名]` 之前加入：

```toml
windows_wsl_setup_acknowledged = true
```

## 保存 API Key

创建 `auth.json`，写入：

```json
{
  "OPENAI_API_KEY": "<SURPLUSTOKEN_API_KEY>"
}
```

将 `<SURPLUSTOKEN_API_KEY>` 替换为你在平台创建的完整密钥。

macOS、Linux 和 WSL 用户建议限制文件权限：

```bash
chmod 600 ~/.codex/auth.json
```

:::warning
`auth.json` 必须是合法 JSON：不能包含注释，最后一个字段后不能有逗号。
:::

也可以让 Codex 从标准输入保存密钥，避免密钥进入命令历史：

```bash
read -rsp "SurplusToken API Key: " SURPLUSTOKEN_API_KEY
echo
printf '%s' "$SURPLUSTOKEN_API_KEY" | codex login --with-api-key
unset SURPLUSTOKEN_API_KEY
```

## 验证配置

进入一个测试目录后启动 Codex：

```bash
mkdir -p ~/codex-test
cd ~/codex-test
codex
```

发送一个不会修改文件的测试请求：

```text
只回复“连接成功”，不要创建或修改任何文件。
```

配置成功应同时满足：

- Codex 能正常返回内容。
- 没有出现鉴权或模型错误。
- SurplusToken 的用量记录中能看到本次请求。

## Codex Desktop

Codex Desktop 使用哪个 `.codex` 目录，取决于当前选择的 Agent Environment：

<SystemContent system="windows">

- 使用原生 Windows 环境时，编辑 `%USERPROFILE%\.codex`。
- 使用 WSL 环境时，编辑对应 WSL 发行版中的 `~/.codex`。
- 打开 WSL 项目时，建议将项目放在 `~/code` 等 Linux 目录中。

</SystemContent>

<SystemContent system="macos">

macOS 使用当前用户的 `~/.codex`。

</SystemContent>

<SystemContent system="linux">

Linux 当前使用 Codex CLI，不需要配置 Desktop。

</SystemContent>

修改配置后，请完全退出并重新打开 Codex Desktop。

## 使用 Cockpit Tools 切换账号

如果以前使用过其他中转服务或 OpenAI 订阅，可以使用 Cockpit Tools 保存多个 Codex 配置并在它们之间切换。

:::warning
Cockpit Tools 是第三方工具，不是 OpenAI 或 SurplusToken 官方组件。安装和使用前请自行确认来源可信，并注意它能够接触你的 API Key。
:::

1. 打开 Cockpit Tools。
2. 进入 **Codex → 账号总览**。
3. 点击 **添加账号**，选择 **API Key**。
4. 供应商选择 **自定义**。
5. 供应商名称填写 `SurplusToken`。
6. API Key 填写平台创建的密钥。
7. 基础地址填写：

   ```text
   https://surplustoken.com/v1
   ```

8. 保存账号，并将它加入 Cockpit Tools 的 **API 服务**。
9. 监听范围选择 **仅本机**，启动本机 API 服务。
10. 选中 SurplusToken 账号，点击 **运行**或**切换 API**。

Cockpit Tools 可能使用类似下面的本机转发地址：

```text
http://localhost:11918/v1
```

这个地址是 Cockpit Tools 的本机代理，不是 SurplusToken 公网地址。不要将监听范围改成 `0.0.0.0`，也不要把本机端口暴露到局域网或公网。

## 常见问题

### 返回 401

通常表示 API Key 填写错误、已撤销或已经过期：

- 重新复制完整密钥，检查前后是否有空格。
- 检查 `auth.json` 是否为合法 JSON。
- 确认修改的是 Codex 当前环境实际读取的 `.codex` 目录。

### 返回 404 或提示模型不存在

依次检查：

- `base_url` 是否以 `/v1` 结尾。
- 是否错误写成了 `/v1/responses`。
- `model` 是否与平台提供的模型 ID 完全一致。
- 当前密钥绑定的分组是否支持该模型。

### 返回 429

可能是请求频率、并发、余额、订阅额度或上游账号受限。请查看平台中的密钥状态、余额和用量记录，稍后重试。

### CLI 可用，但 Desktop 不可用

检查 Desktop 当前选择的是原生 Windows 还是 WSL 环境，并确认修改的是对应环境的配置目录。修改后完全退出并重启 Desktop。

### 修改配置后没有生效

- 完全退出 Codex CLI 或 Desktop 后重新启动。
- 检查文件名是否确实为 `config.toml` 和 `auth.json`。
- Windows 用户确认文件没有被保存成 `config.toml.txt`。
- 检查 TOML 顶层字段是否被误写进 `[model_providers.surplustoken]` 表格。

### Cockpit Tools 切换后仍使用旧账号

- 确认目标账号已加入 **API 服务**。
- 确认本机 API 服务正在运行。
- 再次点击 **运行**或**切换 API**。
- 完全退出并重新启动 Codex。
- 检查本机转发端口是否被其他程序占用。

## 撤销或更换密钥

怀疑 API Key 泄露时：

1. 立即在 SurplusToken 平台撤销旧密钥。
2. 创建新密钥。
3. 更新 `auth.json` 或 Cockpit Tools 中保存的密钥。
4. 重启 Codex，并重新发起测试请求。
