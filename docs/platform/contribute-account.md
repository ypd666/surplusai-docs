# 添加与贡献 OpenAI 账号

贡献账号是指把本人有权使用的 OpenAI/ChatGPT OAuth 账号接入 SurplusToken，由平台按照分组、调度状态和保护策略统一使用。

:::warning 添加后会参与调度
账号创建成功后，默认立即开启“参与调度”。其他符合分组与保护策略的请求可能马上使用该账号，并消耗其上游额度。

如果暂时不希望共享，请在添加成功后立即取消“参与调度”，或选择 **更多 → 暂停共享**。
:::

## 添加前须知

开始前请确认：

- 当前普通用户侧只支持添加 **OpenAI OAuth** 账号，不支持直接添加 API Key。
- 确认账号来源合法，并且账号由本人持有，或已获得明确授权。
- 已了解账号共享可能带来的额度消耗、限流和上游服务条款风险。
- 移除与贡献无关的敏感信息和第三方授权
- 了解贡献期间的使用范围、结算方式和停止规则。
- 确认账号符合平台当前的贡献条件。


不要向他人发送账号密码、验证码、Cookie、Refresh Token、Access Token；完整 OAuth 回调地址也不要对外分享，但需要在 SurplusToken 官方页面回填。

## 第一步：进入账号池

1. 登录 [SurplusToken](https://surplustoken.com)。
2. 在左侧导航进入 **账号池**。
3. 点击页面右上角的 **添加 OAuth 账号**。

弹窗包含两个步骤：

1. **授权方式**
2. **完成授权**

## 第二步：填写账号设置

第一步中按需填写以下字段：

| 页面字段                    | 默认值或约束   | 填写建议                                                 |
| --------------------------- | -------------- | -------------------------------------------------------- |
| **账号名称**                | 必填           | 填写便于自己识别的名称。                                 |
| **平台**                    | OpenAI         | 当前用户侧只支持 OpenAI。                                |
| **账号类型**                | OAuth          | 当前用户侧不支持 API Key 账号。                          |
| **模型白名单**              | 留空           | 留空表示不额外限制模型；填写后只允许所选模型。           |
| **仅允许 Codex 官方客户端** | 默认关闭       | 开启后，普通兼容客户端可能无法使用此账号。               |
| **并发数**                  | 默认 5，最小 1 | 根据账号能力设置；不确定时保持默认。                     |
| **过期时间**                | 默认留空       | 留空表示不设置平台侧过期时间。                           |
| **过期自动暂停调度**        | 默认开启       | 设置过期时间后，到期会自动暂停调度。                     |
| **分组**                    | 默认未选择     | 只能选择当前用户可用且平台为 OpenAI 的分组。             |
| **代理**                    | 默认无代理     | 服务器能直连 OpenAI 时可保持无代理，否则选择并测试代理。 |

:::tip 分组
如果添加后显示 **未绑定分组**，账号已经创建，但还没有进入目标共享分组。请点击 **编辑账号共享设置**，补选一个 OpenAI 分组。
:::

## 第三步：设置贡献保护策略

保护策略用于在上游额度不足前自动停止对其他用户共享。

### 按百分比预留

默认策略为 **按百分比预留**：

- **5h 至少保留**默认是 `0%`。
- **周额度至少保留**默认是 `0%`。
- `0%` 表示不启用该窗口的预留保护，不是“保留全部额度”。

例如设置为 `20%` 时，当对应窗口已使用约 `80%`，平台会停止继续对他人共享。

### 按周预算

按周预算用于限制其他用户在滚动 7 天内产生的消费：

- 自己和共同所有者的使用不计入共享预算。
- 预算为 `0` 表示不设上限。
- 如需限制分享，必须填写一个大于 `0` 的预算。

### 探测失败时

上游额度探测失败时，默认策略是 **继续共享**。还可以选择：

- **暂停共享**
- **按本地账本兜底**

如果百分比预留仍为 `0%`，探测失败策略不会产生预留保护效果。

确认设置后，点击 **下一步**。

## 第四步：完成 OpenAI OAuth 授权

推荐使用默认的 **手动授权**。

1. 点击 **生成授权链接**。
2. 复制生成的链接，在新的浏览器标签页中打开。
3. 登录要贡献的 OpenAI 账号并完成授权。
4. 浏览器最后会跳转到类似下面的地址：

   ```text
   http://localhost:1455/auth/callback?code=...&state=...
   ```

5. 复制浏览器地址栏中的**完整 URL**。
6. 回到 SurplusToken，将完整 URL 粘贴到 **授权链接或 Code**。
7. 点击 **完成授权**。

:::warning 必须复制完整回调 URL
正式流程请不要只复制裸 Code。完整 URL 同时包含 `code` 和 `state`，平台需要二者匹配同一轮授权。

只粘贴 Code 可能出现 `state is required` 或 `invalid oauth state`。
:::

### localhost 页面打不开是否正常？

正常。OAuth 完成后，浏览器会跳到本机回调地址。如果页面显示“无法访问此网站”或“连接被拒绝”，但地址栏已经变成 `localhost:1455/auth/callback?... `，仍可以复制完整地址并回填平台。

OAuth 会话有效期为 30 分钟，并保存在平台服务进程内存中。以下情况需要重新生成授权链接：

- 授权超过 30 分钟。
- 平台服务在授权期间发生重启。
- 回调 URL 来自另一轮授权。
- 生成链接后又点击了重新生成。

## 如何确认添加成功

成功后会提示 **OAuth 账号已添加**，弹窗关闭并刷新账号池列表。

新账号通常应显示：

- 归属：**我的**
- 平台/类型：OpenAI / OAuth
- 状态：**启用**
- 有效状态：通常为 **可用**
- 调度：**参与调度**已开启
- 保护状态：**共享中**，或因保护策略显示 **已停止共享**

:::info 添加成功不等于当前可调度
账号已经出现在列表中，只能说明创建完成。是否能实际处理请求，还取决于调度开关、分组、保护策略、过期时间、代理和上游账号状态。
:::

平台会尽力为贡献者建立私有自用路径，但该过程失败不会阻止账号创建，因此不要把“是否出现自用 Key”作为添加成功的唯一判断。

## 添加后如何管理

本人贡献的账号可以进行以下操作：

- **编辑账号共享设置**：修改模型白名单、分组、代理、并发、过期时间和保护策略。
- **暂停共享 / 恢复共享**：控制账号是否参与调度。
- **测试连接**：检查当前账号与代理是否可用。
- **查看统计**：查看账号的使用情况。
- **定时测试**：配置周期性可用性检测。
- **重新授权**：重新完成 OAuth 授权。
- **刷新令牌**：更新 OpenAI OAuth 令牌。
- **设置隐私**：调整账号隐私选项。
- **恢复状态**：在错误或临时限流恢复后重新启用账号。

其他用户只能看到安全的账号池概览，不能修改你的账号。

## 暂停共享、删除与撤销授权

### 暂停共享

如果只是暂时不希望平台调度该账号，优先使用：

- 取消 **参与调度**
- **更多 → 暂停共享**

### 删除账号

删除操作会把账号从 SurplusToken 中移除并停止平台调度。

:::warning
从 SurplusToken 删除账号，不代表已经在 OpenAI 侧撤销 OAuth 登录或授权。若需要彻底撤权，请同时在 OpenAI 账号侧检查并撤销不再需要的登录会话或授权。
:::

## 其他导入方式

高级用户可能还会看到以下方式：

- **手动输入 RT**：每行一个 OpenAI Refresh Token，可批量验证并创建账号。
- **Codex JSON / AT 批量输入**：每行一个 Token 或 JSON，匹配到已有账号时可能更新原账号。

这些内容均属于高敏感凭据。除非明确了解来源、格式和风险，否则优先使用手动 OAuth 授权，不要把 Token 发到群聊、工单截图或公开文档中。

## 常见问题

| 现象                                        | 原因与处理                                                                                             |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **完成授权**按钮不可用                      | 先生成授权链接并粘贴回调内容；生成或创建仍在进行时按钮也会禁用。                                       |
| localhost 页面打不开                        | 通常是正常现象。确认地址栏已进入 localhost 回调地址，然后复制完整 URL。                                |
| `state is required` / `invalid oauth state` | 粘贴了裸 Code，或回调 URL 与当前授权不是同一轮。重新生成链接并粘贴同一轮完整 URL。                     |
| `session not found or expired`              | OAuth 会话超过 30 分钟，或平台服务发生重启。重新生成授权链接。                                         |
| 服务器无法连接 OpenAI                       | 返回第一步选择可用代理并执行测试，然后重新生成授权链接。代理只作用于平台服务器，不会替浏览器配置网络。 |
| Refresh Token 验证失败                      | Token 可能过期、被撤销、已轮换，或使用了不匹配的导入方式。建议改用手动 OAuth。                         |
| 添加后显示 **不可用**                       | 检查参与调度、保护策略、周预算、过期时间、账号状态和分组，再执行 **更多 → 测试连接**。                 |
| 显示 **未绑定分组**                         | 点击 **编辑账号共享设置**，补选一个可用的 OpenAI 分组。                                                |
| 提示名称、并发、代理或分组错误              | 名称不能为空；并发至少为 1；代理必须存在且启用；分组必须可用且属于 OpenAI 平台。                       |

## 安全建议

- 只在 SurplusToken 官方页面完成授权。
- 不要在截图中显示完整回调 URL、Token、Cookie 或账号 ID。
- 发现异常用量时，立即暂停共享并检查账号统计。
- 不再贡献时，先停止调度，再删除平台账号并检查 OpenAI 侧授权。
- 短时间内不要反复生成链接或重复授权，以免触发上游安全限制。
