# 拼车

拼车用于多人共同承担订阅资源成本，并通过 SurplusToken 的关联分组使用服务。拼车成员不需要互相传递上游账号或个人 API Key。

## 页面包含什么

进入平台左侧的 **拼车** 后，可以在 **拼车广场**和**我的拼车**之间切换，并按照名称、发起人和状态筛选。

每辆车会显示：

- 名称、说明和发起人；
- 服务类型；
- 大车或小车、等级与账号数量；
- 已上车人数、总席位和剩余座位；
- 预计开车时间与加入方式；
- 当前状态和关联分组。

## GPT 拼车的车型与等级

平台会在拼车页面展示当前有效的车型、每级账号数、席位数、基础费用和相对用量计算方式。

- **小车**：每一级增加一个账号和一组较少的席位。
- **大车**：每一级增加一个账号和一组较多的席位。
- **等级**：决定本期拼车包含的账号数与总席位。

开车前需要同时确定车型与等级。确认后当月不再升降级，下个月可以按照届时规则重新选择。

:::warning 以实时规则为准
车型、等级、席位、价格和计算参数可能调整。加入或发起前，请以拼车页面显示的当期规则为准，不要只依赖历史截图或聊天记录。
:::

## 加入拼车

1. 登录 SurplusToken，进入 **拼车 → 拼车广场**。
2. 使用搜索和状态筛选找到目标拼车。
3. 打开详情，确认车型、等级、席位、价格、服务周期和预计开车时间。
4. 确认关联分组支持自己需要的模型和客户端。
5. 阅读退出、退款、续期、封车和解散规则。
6. 页面仍允许加入时，按照提示确认上车和支付。
7. 在 **我的拼车**与**我的订阅**中检查状态。

页面显示“仍有座位”不一定表示当前可以立即加入。例如拼车已经封车、停止招募或等待管理员处理时，应以详情页按钮和状态说明为准。

## 发起拼车

点击页面右上角的 **发起拼车**，按照表单填写服务类型、车型、等级、招募方式、预计时间和说明。

发起前应确认：

- 自己了解当期计费和开车条件；
- 车型和等级能够覆盖预计成员数量；
- 招募说明准确，没有承诺平台规则之外的权益；
- 知道未满员、成员退出、订单取消和续期时如何处理；
- 不会要求成员私下发送密码、验证码、Token 或 API Key。

具体可填写字段和发起资格可能随平台设置变化，请以实际表单为准。

## 拼车状态

| 状态       | 含义                                                           |
| ---------- | -------------------------------------------------------------- |
| **招募中** | 正在寻找成员；能否加入仍取决于剩余座位、加入方式和详情页按钮。 |
| **已封车** | 已停止继续招募，可能正在等待开通或结算。                       |
| **已开车** | 拼车已进入使用阶段，成员应检查订阅与关联分组。                 |
| **已取消** | 本期拼车不再继续，费用和后续处理以订单及平台通知为准。         |

## 开车后如何使用

拼车开通后，平台会为该车关联一个分组。成员通过自己的 SurplusToken API Key 选择该分组，再按照对应客户端文档配置模型。

建议为拼车单独创建密钥，例如 `codex-car1`，这样可以：

- 与普通中转用量分开统计；
- 单独设置额度、速率和有效期；
- 拼车结束后直接禁用或删除该密钥；
- 更容易判断请求是否走了正确的关联分组。

创建方式参见[创建 API 密钥](/platform/create-api-key)和[分组说明](/groups/)。

## 成员注意事项

- 不要把自己的平台密钥发给车主或其他成员；
- 不要索要或传播上游账号密码、验证码、Cookie 和 Token；
- 留意预计开车时间、服务周期、续期和到期通知；
- 定期在使用记录中核对模型、Token 和费用；
- 达到上游额度、并发或平台限制时，请先查看渠道状态和拼车通知；
- 退出、退款或转让席位前，先阅读当前规则并保留订单信息。

## 常见问题

### 上车后为什么还不能使用？

拼车可能仍在招募、已封车但尚未开车，或者订阅与关联分组尚未生效。请同时检查拼车状态、我的订阅、API Key 分组和平台通知。

### 为什么有剩余座位却没有加入按钮？

拼车可能已经封车、停止公开招募，或采用受限的加入方式。页面状态与详情页操作按钮共同决定是否可以加入。

### 拼车结束后密钥还能继续使用吗？

密钥本身可能仍然存在，但关联分组的访问权限可能随订阅结束而失效。建议为拼车使用独立密钥，并在结束后禁用或删除。

### 可以和别人共用一个 API Key 吗？

不建议。每位成员应使用自己的平台账号和 API Key，以便正确统计用量、控制额度和及时撤销。
