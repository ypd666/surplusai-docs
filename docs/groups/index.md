# API 密钥分组

创建 API 密钥时必须选择一个分组。分组会影响可用渠道、适用账号、计费倍率、模型范围和使用限制。

客户端教程只提供推荐分组和接入步骤；具体规则统一放在本目录中维护，并由客户端文档交叉引用，避免同一条分组规则在多个教程中重复出现。

## 分组文档规范

后续每个分组单独建立一个页面，并至少说明：

- 适用的账号或订阅类型；
- 页面显示倍率及计费口径；
- 支持的模型和客户端；
- 并发、速率、额度和有效期限制；
- 适用场景与不适用场景；
- 创建密钥时的选择方法；
- 常见错误与切换分组的方法。
