# 平台介绍

[SurplusToken](https://surplustoken.com) 是基于 Sub2API 开发、面向北大校内师生的 AI 模型接入与资源协作平台。平台把 API 中转、账号共享和订阅拼车整合在同一套账号、密钥、分组、计费与用量系统中，方便用户按自己的需求调用模型或贡献资源。

SurplusToken 提供兼容接口和资源调度服务，不是 OpenAI、Anthropic、Kimi 等模型厂商的官方网站。可用模型、倍率、额度和渠道状态可能随上游及平台规则调整，请以平台实时页面为准。

## 三项核心功能

| 功能         | 解决的问题                                    | 主要使用者               | 使用入口                           |
| ------------ | --------------------------------------------- | ------------------------ | ---------------------------------- |
| **API 中转** | 用统一 API 地址和密钥接入不同模型与编程客户端 | 需要调用模型的普通用户   | API 密钥、可用渠道、使用记录       |
| **账号共享** | 将有权使用的上游账号贡献到账号池并参与调度    | 持有闲置或可共享账号的人 | 账号池、贡献保护、贡献余额         |
| **订阅拼车** | 多人共同承担订阅资源成本并使用关联分组        | 有稳定周期性需求的成员   | 拼车广场、我的拼车、我的订阅与订单 |

这三项功能并不互相排斥。普通用户可以只使用 API 中转；账号持有者可以同时贡献资源；加入拼车后，也仍然通过自己的 SurplusToken API Key 和拼车关联分组接入客户端。

## API 中转

API 中转是平台最基础的使用方式。用户不需要在每个客户端中分别管理不同上游账号，只需创建 SurplusToken API Key，并把客户端指向平台的兼容端点：

```text
https://surplustoken.com/v1
```

一次请求通常经过以下流程：

1. 客户端使用 SurplusToken API Key 发起请求。
2. 平台根据密钥绑定的分组检查模型范围、余额、并发、额度和有效期。
3. 调度系统从当前可用渠道或账号池中选择符合条件的资源。
4. 平台将响应返回客户端，并在使用记录中保存模型、Token、费用、时间和请求状态等信息。

### 中转功能包含什么

- **统一鉴权**：不同客户端可以使用同一 API 体系，也可以为每个客户端单独创建密钥。
- **分组路由**：分组决定可访问的账号类型、渠道、模型、倍率和限制。
- **模型兼容**：可按照平台与分组当前支持情况接入 GPT、Claude、Kimi 等模型。
- **用量与计费**：仪表盘展示余额、请求数、消费、Token、模型分布和最近使用情况。
- **运行状态**：可以查看可用渠道、渠道状态、使用记录和用量统计，便于判断错误来自密钥、余额、分组、渠道还是上游。
- **密钥控制**：密钥可以设置额度、速率、IP 和有效期限制，并可单独禁用或撤销。

开始使用中转服务，请先[创建 API 密钥](/platform/create-api-key)，阅读[分组说明](/groups/)，再选择对应的[客户端接入教程](/clients/codex)。

:::warning 倍率不是模型价格
分组倍率会影响计费，但不等于模型本身的单价。实际扣费还可能受到模型价格、渠道和平台规则影响，请以创建密钥页面与使用记录显示的结果为准。
:::

## 账号共享

账号共享允许用户把本人持有或已获得明确授权的上游账号贡献到 SurplusToken 账号池。账号进入共享调度后，其他符合分组和保护条件的请求可能使用该账号的上游额度。

平台当前的账号池可以展示不同平台和账号类型的资源。普通用户实际能够添加的账号类型，应以“添加贡献账号”页面为准；不要根据账号池中可见的资源推断自己一定拥有对应的导入权限。

### 共享如何运作

1. 贡献者通过平台支持的 OAuth 或其他授权方式添加账号。
2. 贡献者设置分组、模型白名单、并发、到期时间和共享保护策略。
3. 账号满足可用性、调度和保护条件后，进入对应分组的账号池。
4. 平台在请求调度时使用该资源，并记录贡献用量与结算数据。
5. 贡献者可以查看贡献余额，并按照平台当前提供的方式转入账户余额或申请提现。

### 贡献者可以控制什么

- 随时暂停或恢复共享；
- 限制允许调用的模型和并发数；
- 设置 5 小时与周额度的最低预留比例；
- 设置共享周预算；
- 选择额度探测失败时继续共享、暂停共享或使用本地账本兜底；
- 设置到期时间，并在到期后自动暂停调度；
- 查看账号状态、连接测试、统计和贡献余额。

共享并不表示把账号密码或个人 API Key 直接交给其他用户。请求由平台统一调度，其他用户只能看到必要的账号池概览。不过，OAuth 回调地址、Refresh Token、Access Token 和 Cookie 仍属于高敏感凭证，贡献者必须妥善保护。

完整操作步骤和风险说明参见[添加与贡献 OpenAI 账号](/platform/contribute-account)。

## 订阅拼车

拼车面向希望在一段时间内稳定使用订阅资源、又不希望单独承担全部成本的用户。用户可以在拼车广场查看正在招募的车，也可以按平台规则发起新的拼车。

拼车页面会展示：

- 拼车名称、发起人和说明；
- GPT 等服务类型；
- 大车或小车、等级和账号数量；
- 总席位、已上车人数和剩余座位；
- 招募中、已开车、已封车或已取消等状态；
- 预计开车时间、加入方式和关联分组。

当前 GPT 拼车需要在开车前确定车型和等级，确认后当月不再升降级，下个月可按届时规则重新选择。不同车型、等级、席位和费用的计算方式可能调整，请以拼车页面的实时规则为准。

### 加入后如何使用

拼车成功开通后，成员通过拼车对应的关联分组使用服务。成员仍应创建和管理自己的 SurplusToken API Key，不要把个人密钥、上游账号或验证码发送给车主和其他成员。

加入前应确认：

- 当前状态是否仍允许上车；
- 价格、席位、预计开车时间和服务周期；
- 关联分组支持的模型与客户端；
- 退款、退出、续期、封车和解散规则；
- 达到上游额度或平台限制后的处理方式。

详细流程参见[拼车说明](/platform/carpool)。

## 如何选择

| 你的需求                                | 建议方式                                     |
| --------------------------------------- | -------------------------------------------- |
| 偶尔使用模型，希望按实际调用量付费      | 创建 API Key，使用普通中转分组               |
| 需要在 Codex 等客户端中统一管理多个模型 | 使用 API 中转并为不同客户端建立独立密钥      |
| 有本人可合法共享的闲置上游账号或额度    | 添加贡献账号，并设置严格的共享保护策略       |
| 有持续、可预测的订阅使用需求            | 查看拼车广场，对比车型、等级、周期和关联分组 |
| 已加入拼车，同时还需要其他按量模型      | 拼车分组和普通中转分组分别创建密钥           |

## 平台中的其他功能

除三项核心功能外，平台还提供：

- **仪表盘**：查看余额、请求、消费、Token、贡献数据和模型分布；
- **对话**：在网页中直接使用当前可用的模型服务；
- **使用记录与用量统计**：核对每次调用及汇总趋势；
- **渠道与状态监控**：查看可访问渠道、支持模型、定价和运行状态；
- **订阅、订单与充值**：管理订阅、购买记录、余额与兑换码；
- **邀请返利与个人资料**：查看邀请信息并管理账户设置。

## 推荐使用顺序

第一次使用 SurplusToken 时，建议按照以下顺序操作：

1. [注册并登录](/platform/register)。
2. 了解[计费与充值](/platform/billing)以及[密钥分组](/groups/)。
3. [创建 API 密钥](/platform/create-api-key)。
4. 根据需要配置 [Codex](/clients/codex)、[Claude Code](/clients/claude-code)、[Kimi Code](/clients/kimi-code)或 [OpenCode](/clients/opencode)。
5. 在仪表盘和使用记录中确认请求、模型、Token 与费用。
6. 需要贡献资源时阅读[账号共享教程](/platform/contribute-account)；需要稳定订阅资源时阅读[拼车说明](/platform/carpool)。

:::warning 安全与合规
只共享本人持有或已获得明确授权的资源，并遵守学校规定、平台规则和上游服务条款。不要公开 API Key、密码、验证码、Cookie、Token 或完整 OAuth 回调地址。发现异常请求或费用时，应立即禁用相关密钥、暂停共享并检查使用记录。
:::
