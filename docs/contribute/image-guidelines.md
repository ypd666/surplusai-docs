# 文档图片规范

本文记录 SurplusToken 文档中的图片尺寸、标签和存放约定。新增或修改文档图片时，应遵循本规范。

## 统一尺寸

正文中的操作截图统一使用 HTML `img` 标签，并将显示宽度设置为 `720`：

```html
<img
  src="/images/example/example-step.jpg"
  alt="说明当前截图展示的操作"
  width="720"
/>
```

项目主题应为正文图片保留 `max-width: 100%`，因此窄屏设备会按容器宽度缩小，不会产生横向滚动。

不再使用无法统一控制尺寸的 Markdown 图片语法：

```md
![操作截图](/images/example/example-step.jpg)
```

## 使用规则

- 普通操作截图统一设置 `width="720"`。
- 需要展示更小的图标、徽标或局部控件时，可以使用更小宽度，但应在相关文档中保持一致。
- 不要把小尺寸图片放大到 `720`，避免画面模糊；应重新截取清晰原图。
- `alt` 必须准确说明图片内容，不能使用“截图 1”“图片”等无意义描述。
- 图片本身不要包含完整 API Key、密码、邮箱、余额、真实姓名或其他敏感信息。
- 同一操作在多篇文档中出现时，复用同一图片，不要保存重复副本。

## 文件格式

- 界面截图优先使用 `.jpg` 或经过压缩的 `.png`。
- 需要透明背景的图标使用 `.png` 或 `.svg`。
- 文件扩展名必须与实际图片格式一致。
- 文件名使用小写英文和连字符，例如 `select-plus-team-group.jpg`。

## 存放位置

文档图片统一放在：

```text
docs/public/images/<主题>/
```

例如 Codex 教程图片放在：

```text
docs/public/images/codex/
```

文档中使用从站点根目录开始的路径：

```html
<img
  src="/images/codex/select-plus-team-group.jpg"
  alt="选择 plus/team 分组"
  width="720"
/>
```

不要在文档中使用本机绝对路径、`file://` 地址或作者个人目录路径。

## 提交前检查

提交图片相关修改前，至少确认：

1. 浅色与深色主题下图片内容均可辨认。
2. 桌面端图片宽度一致。
3. 手机或窄窗口下图片不会溢出正文区域。
4. 图片不包含密钥或个人信息。
5. 图片路径、扩展名和实际文件格式一致。
6. 所有图片都具有有意义的 `alt` 文本。
